## 🧠 PROMPT START

Build a full-stack web application called **“KudoBoard”** — a lightweight, server-rendered employee appreciation board with real-time updates and PDF export capability.

Users can:

* View events
* Submit kudos/messages
* See real-time updates
* Export all messages for an event as a **beautiful PDF**

## Backend

* Python 3.11+
* FastAPI
* Jinja2 templates
* SQLite (SQLAlchemy ORM)
* WebSockets for real-time updates

## Frontend

* Server-rendered HTML
* Tailwind CSS
* Alpine.js
* HTMX (optional)

## PDF Generation (IMPORTANT)

Use:

* **WeasyPrint (preferred)** OR
* ReportLab (fallback)

WeasyPrint is preferred because it renders HTML → PDF with CSS styling.

# 📦 PDF EXPORT FEATURE

---

## 🎯 Goal

Allow users to export **all kudos/messages for a specific event** into a well-designed PDF.

---

## 🔌 New Endpoint

### GET `/event/{event_id}/export`

Behavior:

* Fetch all messages for event
* Render HTML template
* Convert HTML → PDF
* Return as downloadable file

---

## 🧾 PDF Content Requirements

The PDF should include:

### Header Section

* Event Name
* Event Description
* Event Image (optional banner)
* Export date/time

---

### Message Section

For each message:

* Name (bold)
* Message content (render rich text safely)
* Theme styling (subtle visual differentiation)
* Timestamp

---

### Footer

* “Generated by KudoBoard”
* Page numbers

---

## 🎨 PDF Design Guidelines

* Clean and professional
* Use Tailwind-like styling OR embedded CSS
* Ensure print-friendly layout:

  * Good spacing
  * No overflow issues
* Support multi-page content
* Avoid dark backgrounds (print-friendly)

---

## 📄 HTML Template for PDF

Create:

```id="pdfhtml1"
templates/pdf/event_pdf.html
```

This template should:

* Be separate from UI templates
* Use inline or embedded CSS
* Be optimized for print (A4 size)

---

## ⚙️ PDF Service Layer

Create:

```id="pdfservice1"
app/services/pdf_service.py
```

Responsibilities:

* Load event + messages
* Render Jinja template → HTML
* Convert HTML → PDF
* Return binary stream

---

## 🧪 Example Implementation Flow

1. User clicks **“Export as PDF”** button on event page
2. Frontend calls:

   ```
   GET /event/{event_id}/export
   ```
3. Backend:

   * Fetch data
   * Render HTML
   * Generate PDF
   * Return response with:

     ```
     Content-Type: application/pdf
     Content-Disposition: attachment; filename="event-name.pdf"
     ```

---

## 🖱️ UI Changes

On Event Page:

* Add button:

  * “📄 Export as PDF”
  * Positioned near event title or top-right

Optional:

* Loading indicator while generating

---

## 🔒 Security & Validation

* Validate event_id exists
* Escape unsafe HTML before rendering PDF
* Prevent injection in PDF template

---

## ⚡ Performance Considerations

* Handle large number of messages (pagination optional)
* Optimize image loading (optional: disable images in PDF if slow)

---

## 🧪 Testing

* Test PDF generation with:

  * No messages
  * Few messages
  * Large dataset (100+ messages)

---

## 🚀 Deployment Notes

If using WeasyPrint:

* Ensure system dependencies installed:

  * libpango
  * libcairo
  * libgdk-pixbuf

Provide install instructions for:

* macOS
* Ubuntu (VPS)
